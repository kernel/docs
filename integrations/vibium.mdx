[Vibium](https://github.com/VibiumDev/vibium) is a brand-new browser automation framework for AI agents that is built on WebDriver BiDi, a W3C standard. With their [latest release](https://github.com/VibiumDev/vibium/releases/tag/v26.3.9), any agent can now use Vibium to connect to Kernel and navigate pages, fill forms, click buttons, and take screenshots across 1 or 10,000 cloud browsers running in parallel. We're really excited to keep working with Jason Huggins and to keep supporting open web standards.

## Adding Kernel to existing Vibium implementations

If you already have a Vibium implementation, you can switch to using Kernel's cloud browsers by connecting to Kernel's CDP endpoint with authentication.

### 1. Install the Kernel SDK

<CodeGroup>
```bash npm
npm install @onkernel/sdk
```

```bash pip
pip install kernel
```
</CodeGroup>

### 2. Install Vibium

<CodeGroup>
```bash npm
npm install vibium
```

```bash pip
pip install vibium
```
</CodeGroup>

### 3. Initialize Kernel and create a browser

<CodeGroup>
```typescript TypeScript
import Kernel from "@onkernel/sdk";
import { browser } from "vibium";

const kernel = new Kernel();

const kernelBrowser = await kernel.browsers.create({ stealth: true });

console.log("Live view url: ", kernelBrowser.browser_live_view_url);
```

```python Python
from onkernel import Kernel
from vibium.sync_api import browser

kernel = Kernel()

kernel_browser = kernel.browsers.create(stealth=True)

print("Live view url: ", kernel_browser.browser_live_view_url)
```
</CodeGroup>

### 4. Connect Vibium to your Kernel browser

Connect Vibium to Kernel's BiDi WebSocket endpoint with authentication:

<CodeGroup>
```typescript TypeScript
const bro = await browser.start(kernelBrowser.cdp_ws_url, {
  headers: { Authorization: `Bearer ${process.env.KERNEL_API_KEY}` },
});

const vibe = await bro.page();
```

```python Python
bro = browser.start(kernel_browser.cdp_ws_url, headers={
    "Authorization": f"Bearer {os.environ['KERNEL_API_KEY']}",
})

vibe = bro.page()
```
</CodeGroup>

### 5. Use your Vibium automation

Use Vibium's page methods with the Kernel-powered browser:

<CodeGroup>
```typescript TypeScript
await vibe.go("https://kernel.sh");
console.log(await vibe.title());
console.log(await vibe.find("h1").text());

// Clean up
await bro.stop();
await kernel.browsers.deleteByID(kernelBrowser.session_id);
```

```python Python
vibe.go("https://kernel.sh")
print(vibe.title())
print(vibe.find("h1").text())

# Clean up
bro.stop()
kernel.browsers.delete_by_id(kernel_browser.session_id)
```
</CodeGroup>

## Using Vibium CLI with Kernel

You can also use Vibium's CLI with Kernel by setting environment variables:

```bash
export VIBIUM_CONNECT_URL=<your-kernel-cdp-ws-url>
export VIBIUM_CONNECT_API_KEY=<your-kernel-api-key>

vibium go https://example.com
vibium title        # "Example Domain"
vibium text h1      # "Example Domain"
```

## Using Vibium MCP server with Kernel

The MCP server reads the same environment variables, so AI agents can use a Kernel-powered browser:

```bash
VIBIUM_CONNECT_URL=<your-kernel-cdp-ws-url> \
VIBIUM_CONNECT_API_KEY=<your-kernel-api-key> \
vibium mcp
```

Or in your Claude Desktop / Claude Code config:

```json
{
  "mcpServers": {
    "vibium": {
      "command": "vibium",
      "args": ["mcp"],
      "env": {
        "VIBIUM_CONNECT_URL": "<your-kernel-cdp-ws-url>",
        "VIBIUM_CONNECT_API_KEY": "<your-kernel-api-key>"
      }
    }
  }
}
```

## Benefits of using Kernel with Vibium

- **No local browser management**: Run automations without installing or maintaining browsers locally
- **Scalability**: Launch multiple browser sessions in parallel
- **Stealth mode**: Built-in anti-detection features for web scraping
- **Session state**: Maintain browser state across runs via [Profiles](/profiles/overview)
- **Live view**: Debug your automations with real-time browser viewing
