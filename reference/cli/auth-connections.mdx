---
title: "Auth Connections"
---

Manage [managed auth](/auth/configuration) connections that keep browser profiles logged into a domain. A connection drives a login flow once, then keeps the profile's session healthy over time.

## `kernel auth connections list`
List managed auth connections.

| Flag | Description |
|------|-------------|
| `--domain <domain>` | Filter by domain. |
| `--profile-name <name>` | Filter by profile name. |
| `--limit <n>` | Maximum number of results to return. |
| `--offset <n>` | Number of results to skip. |
| `--output json`, `-o json` | Output raw JSON array. |

## `kernel auth connections get <id>`
Get a managed auth connection by ID.

| Flag | Description |
|------|-------------|
| `--output json`, `-o json` | Output raw JSON object. |

## `kernel auth connections create`
Create a managed auth connection for a profile and domain.

| Flag | Description |
|------|-------------|
| `--profile-name <name>` | Name of the profile to manage. Required. |
| `--domain <domain>` | Target domain for authentication. Required. |
| `--login-url <url>` | Optional login page URL to skip discovery. |
| `--allowed-domain <domain>` | Additional allowed domains (repeatable). |
| `--health-check-interval <seconds>` | Interval between health checks (`300`–`86400`). |
| `--proxy-id <id>` | Proxy ID to use. |
| `--proxy-name <name>` | Proxy name to use. |
| `--credential-name <name>` | [Kernel credential](/reference/cli/credentials) name to use. |
| `--credential-provider <name>` | External [credential provider](/reference/cli/credentials#credential-providers) to look up credentials from. |
| `--credential-path <path>` | Provider-specific path (e.g. `VaultName/ItemName`). |
| `--credential-auto` | Look up by domain from the specified provider (defaults to true when `--credential-provider` is set without `--credential-path`). |
| `--no-save-credentials` | Don't save credentials after a successful login. |
| `--output json`, `-o json` | Output raw JSON object. |

```bash
kernel auth connections create --profile-name my-profile --domain example.com
```

## `kernel auth connections update <id>`
Update a managed auth connection.

| Flag | Description |
|------|-------------|
| `--allowed-domain <domain>` | Additional allowed domains (replaces existing list). |
| `--login-url <url>` | Login page URL (set to empty string to clear). |
| `--health-check-interval <seconds>` | Interval between health checks. |
| `--proxy-id <id>` | Proxy ID to use. |
| `--proxy-name <name>` | Proxy name to use. |
| `--credential-name <name>` | [Kernel credential](/reference/cli/credentials) name to use. |
| `--credential-provider <name>` | External [credential provider](/reference/cli/credentials#credential-providers) to look up credentials from. |
| `--credential-path <path>` | Provider-specific path (e.g. `VaultName/ItemName`). |
| `--credential-auto` | Look up by domain from the specified provider. |
| `--save-credentials` | Enable saving credentials after a successful login. |
| `--no-save-credentials` | Disable saving credentials after a successful login. |
| `--output json`, `-o json` | Output raw JSON object. |

## `kernel auth connections login <id>`
Start a login flow for a connection.

| Flag | Description |
|------|-------------|
| `--proxy-id <id>` | Proxy ID to use for this login. |
| `--proxy-name <name>` | Proxy name to use for this login. |
| `--output json`, `-o json` | Output raw JSON object. |

## `kernel auth connections submit <id>`
Submit field values to an in-progress login flow.

| Flag | Description |
|------|-------------|
| `--field <name=value>` | Field name/value pair (repeatable). |
| `--mfa-option-id <id>` | MFA option ID if the user selected an MFA method. |
| `--sign-in-option-id <id>` | Sign-in option ID if the flow returned non-MFA choices. |
| `--sso-provider <provider>` | SSO provider if the user chose an SSO button (e.g. `google`, `github`). |
| `--sso-button-selector <xpath>` | XPath selector if the user chose an SSO button. |
| `--output json`, `-o json` | Output raw JSON object. |

## `kernel auth connections follow <id>`
Follow login flow events for a connection.

| Flag | Description |
|------|-------------|
| `--output json`, `-o json` | Output raw JSON object. |

## `kernel auth connections delete <id>`
Delete a managed auth connection.

| Flag | Description |
|------|-------------|
| `--yes`, `-y` | Skip confirmation prompt. |
